# Slide Outline - CMAPSS PHM Presentation

## Deck Objective
Present the CMAPSS PHM project as a complete predictive-maintenance AI system that integrates:
- calibrated RUL prediction
- deterministic risk and recommendation logic
- scenario-based what-if analysis
- dashboard-centered traceability and auditability

## Audience
- thesis / academic evaluation committees
- project supervisors
- engineering reviewers
- PHM / AI demo audiences

## Time Budget
- main talk: `12-15 minutes`
- backup / Q&A: `5-10 minutes`

## Slide Sequence
1. `Title`
2. `Motivation and Problem`
3. `NASA C-MAPSS Context`
4. `Project Objectives and Contributions`
5. `System Architecture`
6. `Data and Preprocessing`
7. `Predictive Layer and Champion Selection`
8. `Agent Layer and Deterministic Decision Logic`
9. `Dashboard Layer and User Workflow`
10. `Experiments and Results`
11. `Scenarios and Interpretability`
12. `Engineering Constraints`
13. `Limitations and Future Work`
14. `Conclusion`
15. `Backup / Appendix`

## Main Storyline
The presentation should move from problem motivation to full-system evidence:
1. explain why PHM needs more than point RUL prediction
2. show the benchmark and the engineering problem
3. explain the three-layer architecture
4. justify the deployed champion model
5. show how the agent layer transforms prediction into decision support
6. demonstrate the dashboard as the integrated artifact
7. present evaluation, scenario behavior, and engineering tradeoffs
8. conclude with the scientific and engineering contribution

## Slide Design Principles
- one message per slide
- use metrics only when traceable to project artifacts
- keep summary slides executive and analysis slides evidence-backed
- prefer visuals from the implemented app and generated metrics over conceptual filler
- keep backup material separate from the main narrative

## Exit Criteria
- Each main slide can be explained in under one minute.
- Every result or claim maps to a real artifact.
- The deck remains coherent even without a live demo.
